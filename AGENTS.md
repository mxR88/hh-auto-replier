# hh-auto-replier

Автоматический поиск и отклик на вакансии HeadHunter под резюме Максима Руденко (Linux/DevOps/SRE).

## Проект

- `src/main.py` — поиск и фильтрация вакансий
- `src/reply.py` — генерация сопроводительного письма и отправка отклика
- `src/vacancy_info.py` — получение и вывод описания вакансии (с очисткой HTML через BeautifulSoup)
- `src/config.py` — конфигурация (API ключи, фильтры поиска)
- `src/hh/client.py` — HTTP-клиент для API HH
- `src/hh/models.py` — модели данных вакансии
- `src/hh/matcher.py` — загрузка навыков из резюме, фильтрация и скоринг
- `resume.html` — резюме в формате HTML (скачано с HH)
- `seen_vacancies.txt` — ID вакансий, которые уже показаны (автоматически пополняется)
- `.env` — секреты (не коммитить!)

## Команды

```sh
just scrape          # поиск подходящих вакансий
just reply <url>     # сгенерировать и отправить отклик
just info <url>      # показать описание вакансии (с очисткой HTML)
just pop             # взять следующую нерассмотренную вакансию из all_vacancies.json (удаляет из очереди, добавляет в seen)
just skip <id/url>   # пропустить вакансию (добавить ID в seen_vacancies.txt)
```

## Конфигурация

Все настройки в `src/config.py`:
- `search_text` — поисковый запрос
- `search_area` — регионы (1 = Москва, 113 = Россия)
- `exclude_words` — слова-исключения (Windows, Microsoft, 1C и т.д.)
- `salary_from` — минимальная зарплата
- `search_schedule` — график (remote, fullDay, flexible)

## API HH

Токен авторизации (user OAuth) для откликов хранится в `.env`.
При отклике через `just reply` — генерируется письмо, выводится в stdout.
Отправка через HH API не реализована — отклик отправляется вручную через браузер
по ссылке `apply_alternate_url` из вакансии.

Соответствие навыков считается по вхождению слов из резюме в название, требования,
описание и ключевые навыки вакансии. Слова из `exclude_words` фильтруются полностью.

## Cover letter

**ВАЖНО:** Сопроводительное письмо генерируется под каждую вакансию индивидуально.
Агент читает полное описание вакансии (description, requirement, responsibility, key_skills)
и подбирает релевантные навыки из резюме. Письмо должно быть персонализированным —
упоминать конкретные технологии из вакансии, а не быть шаблонным.

## Очередь вакансий

`all_vacancies.json` — полный список найденных вакансий. `just pop` берёт следующую
непросмотренную, удаляет из JSON и добавляет ID в `seen_vacancies.txt`. Нерелевантные
вакансии пропускаются одним `just pop` (без `just info`).

При отклике через `just pop` + `just info <url>`:

1. Агент получает следующую вакансию из очереди через `just pop`
2. Если нужно — читает детали через `just info <url>`
3. Анализирует требования, описание и ключевые навыки
4. Подбирает пересекающиеся навыки из резюме
5. Пишет персонализированное сопроводительное письмо (сам, не через скрипт)
6. Выводит ссылку и письмо. Пользователь открывает в браузере и отправляет вручную.

## Cover letter — правила для агента (opencode)

**Это ключевой момент работы.** Агент (opencode) не просто запускает скрипт, а:

1. **Берёт следующую вакансию из очереди** — через `just pop` получает URL, название и компанию. Если название/компания заведомо нерелевантны (AI/ML, embedded, C++, Windows, 1С, менеджмент) — сразу `just pop` дальше.
2. **Читает описание вакансии вручную** — через `just info <url>` (или напрямую через `src/vacancy_info.py`) получает чистый текст description, requirement, responsibility, key_skills с очисткой HTML через BeautifulSoup.
3. **Оценивает релевантность** — если вакансия про embedded/C++/OpenBMC/драйверы — пропускает, это не профиль Максима.
3. **Пишет живое, персонализированное письмо** — не шаблон. Упоминает конкретные технологии из вакансии, которые пересекаются с опытом Максима. Показывает, что кандидат читал описание.
4. **Формат работы:** агент читает вакансию через API, пишет письмо сам (не через скрипт), выводит ссылку и письмо. Пользователь открывает в браузере и отправляет вручную.
5. **Тон письма:** живой, конкретный, без воды. Показать что кандидат вник в требования. Не копировать шаблон.
6. **Пропускать нерелевантные вакансии** — если вакансия про embedded/C++/OpenBMC/драйверы/низкоуровневую разработку — не тратить время.

## Важно

- Не коммитить `.env` с токенами
- User-Agent: `HH-Rust-Bot/1.0 (https://t.me/rust_hh_jobs_bot)`
- Для `GET /vacancies` используется app token (client_credentials), его можно
  получить в рантайме. Для откликов (applicant) нужен user OAuth token.
- При поиске вакансии, на которую уже откликнулись, не показывается (HH скрывает
  её из результатов для данного пользователя).
